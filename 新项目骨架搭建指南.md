## 构建模式规则

### 进入条件
AI 第一次读取本文件时，自动进入构建模式。**以下规则覆盖 project_rules.md 的"先沟通后执行"原则**——当前场景下，提问本身就是"沟通"，生成文件就是"执行"。

### 行为规则
1. **先问后建** — 每步先向用户提问关键决策，等确认后再创建文件
2. **用完即走** — 完成最后一步后，自动退出构建模式，进入正常开发模式
3. **留印** — 每一步完成后写入 `sessions/{日期}/_build.md` 日志
4. **断点续建** — 任何时候中断，下次对话再读本文件时，先读 `_build.md` 恢复进度
5. **跳过原则** — 用户说"默认"或"跳过"时，使用本文档提供的默认值，不追问
6. **输出原则** — 每步生成文件后告知用户"已创建 → {文件路径}"，不啰嗦

### 构建完成后
- 在 `solo_session_context.json` 标记 `_built: true` 和 `_buildTime`
- 告诉用户项目骨架已就绪，退出构建模式

## 附录 B — 核心设计模式速查

### B1. 身份前缀系统

```
开发模式：
  「开发：」正文      → 开发工程师正常工作
  「开发监督：」正文   → 发现问题需要回应
  「总导演：」正文     → 总导演/创作者正常工作
  「创作监督：」正文   → 发现问题需要回应
  无前缀              → ❌ 掉身份

修复规则：回复永远不能没有前缀
```

### B2. 监督管道架构

```
用户输入
  ↓
isReadOnlyQuery()  ← 查询/搜索类直接放行
  ↓（写操作）
前置校验 → checklist-before.json → 通过则执行
  ↓
执行操作
  ↓
后置校验 → checklist-after.json → 附提醒完成
```

### B3. 记忆分区系统

```
solo_session_context.json
  ├── _active: "tools" 或 "story"
  ├── tools: { last_topic, decisions[], pending[] }
  ├── story: { last_topic, decisions[], pending[] }  ← 含轮次计数器
  └── _supervisorNotes: { tools: [], story: [] }

切换规则：说"切xxx模式" → 读取对应分区 → 加载对应角色
会话日志：开发模式 → sessions/{日期}/_tools.md
          创作模式 → sessions/{日期}/{项目名}.md
```

### B4. SKILL 六层架构

```
┌───────────────────────────────────────────────┐
│  身份监督层   开发监督 / 创作监督               │
│  (谁在说话？) 控制前缀/自检/出声时机            │
├───────────────────────────────────────────────┤
│  AGENT 层     developer-engineer (可选)        │
│  (谁在干活？) DeepSeek 编码子 AGENT             │
├───────────────────────────────────────────────┤
│  内容管线层   推送看板/融合触发等                │
│  (创作产出)   看板同步/规则提炼/内容管道         │
├───────────────────────────────────────────────┤
│  审计修复层   循环自检 / 深度审计               │
│  (代码质量)   扫描→修复→排序→决策               │
├───────────────────────────────────────────────┤
│  数据上下文层  记忆切换/读项目/资料库            │

├───────────────────────────────────────────────┤
│  维护工具层   备份/数据修复/规则审查/标准化      │

└───────────────────────────────────────────────┘
```

### B5. 7 步编码工作流

```
① 读脑图+索引 → ② 读目标函数前后30行 → ③ 讨论方案
→ ④ 执行修改 → ⑤ 运行验证 → ⑥ 更新脑图+索引
→ ⑦ 写入 sessions 日志
```

### B6. 文档三层架构

```
规则层：project_rules.md         ← AI 行为底线，每次对话自动注入
手册层：docs/手册/*.md           ← 业务架构/接口/流程详情，按需加载
                                 ← 超过 50 行的说明必须拆入此层
索引层：code_index.json          ← @函数名 → 文件路径，自动扫描
                                 ← 改完代码跑 node scan_index.js
```

### B7. 数据三区架构

```
全局数据  → data/__global__/     ← 跨模块共享
模块数据  → data/{模块}/data/    ← 每个独立模块
          → data/{模块}/presets/ ← 预设模板
版本快照  → .versions/{模块}/    ← 回滚和追溯
```

### B8. 会话日志格式

```markdown
--- 会话起始 | {YYYY-MM-DD HH:MM} ---
你：「用户原话」
{HH:MM} [{标签}] → 执行摘要
```

### B9. 数据文件类型分类

| 类型 | 例子 | 特性 |
|:----|:-----|:-----|
| 配置型 | `settings.json` | 几乎不改，手动编辑 |
| 状态型 | `state.json` | 频繁读写，需要版本化 |
| 产出型 | `output.json` | 生成后基本不改，版本化 |
| 索引型 | `index.json` | 自动生成，只读引用 |
| 临时型 | `_temp.json` | 可随意删除 |
