## 执行步骤

### Step 1 — 创建会话与备份目录

**做什么：** 创建 `sessions/` 和 `backups/` 目录骨架。

```
sessions/
  {YYYY-MM-DD}/
    session.md          ← 当天会话日志（通用）
    方案讨论.md          ← WORK 模式的对话记录（可选）
    开发记录.md          ← CODE 模式的对话记录（可选）

backups/
  README.md             ← 简单说明：手动备份放这里
```

**输出格式：** `sessions/{日期}/session.md` 统一用以下格式：

```markdown
--- 会话起始 | {YYYY-MM-DD HH:MM} ---
你：「用户说的话」
{HH:MM} [{标签}] → 做了什么

标签池：[需求变更] [定稿完成] [代码生成] [Bug修复] [架构变更] [规则入库] [验证通过]
```

### Step 3 — 创建 SKILL 模板文档

**做什么：** 创建标准化 SKILL 模板，供以后新增 SKILL 时参考。

**路径：** `.trae/skills/SKILL模板.md`

```markdown

description: "<一句话描述：核心功能 + 触发条件>"

## 1. 归属模块
<所属模块名>

## 2. 核心用途
<1-3 句话说明做什么、产出什么>

## 3. 触发条件
- **核心触发词**：用户说什么话时触发
- **同义说法**：其他等效说法
- **不触发场景**：什么时候不要触发

## 4. 能力边界
### 能做
1. <能力1>
2. <能力2>
### 不能做
1. <红线1>
2. <红线2>

## 5. 调用后行为
<调用后的效果、输出格式、数据落盘位置>
```

### Step 4 — 创建「读项目」SKILL

**做什么：** 让任意模式可以快速读取项目概况，不用手动翻文件。

**路径：** `.trae/skills/read-context/SKILL.md`

```markdown
---
name: "读项目"
description: "读取项目规则、脑图和会话记录，了解项目架构和当前状态。用户说读项目/加载项目/回顾项目时触发。"
---

# read-context — 读取项目上下文

## 1. 归属模块
上下文管理

## 2. 核心用途
读取 `project_rules.md` + `脑图.md` + 最近会话记录，输出项目概况。

## 3. 触发条件
- **核心触发词**：读项目、读规则、加载项目、回顾项目
- **不触发场景**：正常对话中

## 4. 能力边界
### 能做
1. 读取 project_rules.md
2. 读取 脑图.md
3. 读取 docs/方案/ 下最近的方案文件
### 不能做
1. 不能修改任何文件

## 5. 调用后行为
结合规则、脑图、最近方案，输出项目概况总结。
```
