## 脑图 B：三模式关系（真实版）

三个模式的对话**完全不互通**，各管各的记忆。信息传递靠文件 + 用户口头转述。

```mermaid
flowchart LR
    subgraph 三个独立AI
        WORK[WORK 辅助规划]
        CODE[CODE 核心主力]
        DESIGN[DESIGN 辅助美化]
    end

    User([你]) -->|切模式| WORK
    User -->|切模式| CODE
    User -->|切模式| DESIGN

    WORK -->|输出方案文件| File[docs/方案/ 目录]
    File -.->|你告诉我文件名| CODE
    File -.->|你告诉我文件名| DESIGN

    style WORK fill:#f5a623,color:#fff
    style CODE fill:#4a90d9,color:#fff
    style DESIGN fill:#7ed321,color:#fff
    style User fill:#e74c3c,color:#fff
```

## 信息传递方式

```mermaid
flowchart TD
    Start[你找WORK聊方案] -->|聊完了| Out[WORK输出方案文档]
    Out -->|存在| Doc[docs/方案/YYYY-MM-DD_主题.md]
    Doc -->|你切到CODE 告诉我文件名| CodeRead[CODE读取文档]
    Doc -->|你切到DESIGN 告诉我文件名| DesignRead[DESIGN读取文档]
    CodeRead -->|直接开干| Done
    DesignRead -->|开始美化| Done

    subgraph 也可以口头传
        Verbal[你直接告诉新模式 WORK聊了什么]
    end

    style Doc fill:#eee,color:#333
```

## OpenViking 定位

OpenViking 是**快速检索工具**，不是跨模式信息传递工具。

```
✅ 适用：搜索某条设定在哪、找代码位置、快速查信息
❌ 不适用：存方案、传对话、做跨模式沟通（用文件更可靠）
```
