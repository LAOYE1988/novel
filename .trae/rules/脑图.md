## 脑图 B：三模式协作关系

```mermaid
flowchart TD
    User[用户提需求] --> CODE

    subgraph 核心主力
        CODE[CODE 工程师]
    end

    subgraph 辅助增强
        WORK[WORK 规划师]
        DESIGN[DESIGN 美工]
    end

    CODE -->|需要规划方案| WORK
    CODE -->|需要美化界面| DESIGN
    WORK -->|输出方案文档| CODE
    DESIGN -->|输出UI| CODE

    CODE -.->|没有WORK/DESIGN也能独立做| Finish[完成功能]
    WORK -.->|可被跳过| CODE
    DESIGN -.->|可被跳过| CODE

    style CODE fill:#4a90d9,color:#fff
    style WORK fill:#f5a623,color:#fff
    style DESIGN fill:#7ed321,color:#fff
```

## 三模式职责明细

```mermaid
flowchart LR
    subgraph CODE职责
        C1[制作工具功能]
        C2[修改BUG]
        C3[自主技术决策]
        C4[需要时叫WORK/DESIGN]
    end

    subgraph WORK职责
        W1[功能规划 结构设计]
        W2[输出方案文档]
        W3[梳理需求]
        W4[不写代码 不做UI]
    end

    subgraph DESIGN职责
        D1[界面美化]
        D2[视觉优化]
        D3[不参与功能决策]
        D4[需要时才介入]
    end
```

## 模式切换注意事项

```
模式之间对话不互通
但三份模式共享同一份顶层规则（project_rules.md）

当 CODE 中途需要 WORK 协助时：
  1. CODE 需说明当前进度和遇到的问题
  2. WORK 先读 project_rules.md 了解项目背景
  3. 再基于 CODE 的说明给出方案建议

当 CODE 需要 DESIGN 协助时：
  1. CODE 说明需要美化的具体界面和功能
  2. DESIGN 只做视觉优化，不改功能逻辑
```
