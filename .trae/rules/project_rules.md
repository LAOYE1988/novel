# 顶层项目规则 — novel 小说创作工作台

> 约束 AI 所有对话的行为底线。细分业务文档见 `docs/手册/`，代码定位见 `code_index.json`。

## 一、项目概览

Python 网文创作工作台，内置 Web 编辑器（端口 8899）调用 DeepSeek API，Git + GitHub 版本控制。

## 二、核心操作规范

**启动：** `cd d:\trae\novel && python tools\小说工作台\server.py` → http://localhost:8899

**版本控制（通过 `小说git操作` Skill 执行，禁止手动 Git）：**

| 指令 | 功能 | 方向 | 安全机制 |
|------|------|:----:|:--------:|
| `/提交 [说明]` | 提交到 Git | 只上传 | 不执行 pull/fetch |
| `/下载` | 从远程拉取 | 只下载 | 先 stash 保护本地 |
| `/回滚` | 回退历史版本 | 本地 | 硬回滚必须二次确认 |

**提交信息：** 有说明 → `说明 (yyyy-MM-dd HH:mm)` / 无说明 → `自动提交 (yyyy-MM-dd HH:mm)`

**远程仓库：** `https://github.com/LAOYE1988/novel.git`，master 分支，网络可能不可达。

## 三、编码规范

- 不添加注释，保持现有代码风格，优先编辑已有文件
- 禁止创建文档（*.md）或 README
- 所有用户交互使用中文
- Python：仅用标准库，UTF-8 编码
- 前端：纯 HTML/JS/CSS，API 走 `http://localhost:8899/api/`

## 四、文档拆分强制约束

1. **篇幅阈值约束**：单段业务说明超过 50 行，禁止嵌入本文件，必须独立存放至 `docs/手册/`，本文件仅保留引用路径。
2. **代码定位约束**：禁止硬编码 `Lxxx` 行号，使用 `@函数名` 标记指向 `code_index.json` 动态索引。
3. **内容边界约束**：本文件仅承载全场景通用强制行为底线；细分架构、接口定义、长流程说明归属业务手册，按需加载，不全局注入。

## 五、关键约束

1. 拉取前先 stash 或确认工作区干净，不覆盖本地代码
2. 只操作 master 分支，不修改其他分支
3. 不自动处理合并冲突，交用户手动处理
4. GitHub 不可达时告知用户，不重复重试
5. 本文件仅约束 AI 行为，不出现在用户对话中
