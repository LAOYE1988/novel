# 顶层项目规则 — novel 小说创作工作台

> 本文档定义本项目的核心规则、架构概览和操作规范。任何 AI 代理或新上手开发者请先阅读此文件。

---

## 一、项目概览

**项目名称：** 小说创作工作台（Novel Writing Workbench）

**项目定位：** 网文小说创作辅助工具，集成 AI 智能体对话、创作看板、多项目管理、版本控制为一体的工作台。

**技术栈：**
- 后端：Python 标准库（`http.server`），无需安装第三方包
- 前端：纯 HTML + JavaScript + CSS，单页应用
- AI 接口：DeepSeek API（`deepseek-chat` 模型）
- 版本控制：Git + GitHub 远程仓库
- 启动端口：8899

## 三、核心操作规范

### 3.1 启动项目

```bash
cd d:\trae\novel
python tools\小说工作台\server.py
```

启动后访问 http://localhost:8899

### 3.2 版本控制（通过 Skill 执行）

所有 Git 操作通过 `小说git操作` Skill 执行，禁止手动运行 Git 命令：

| 指令 | 功能 | 方向 | 安全机制 |
|------|------|:----:|:--------:|
| `/提交 [说明]` | 提交本地修改到 Git | 只上传 | 不执行 pull/fetch |
| `/下载` | 从远程拉取最新代码 | 只下载 | 先 stash 保护本地 |
| `/回滚` | 回退到指定历史版本 | 本地操作 | 硬回滚必须二次确认 |

**提交信息格式：**
- 用户提供说明 → `说明 (yyyy-MM-dd HH:mm)`
- 用户未提供说明 → `自动提交 (yyyy-MM-dd HH:mm)`

### 3.3 远程仓库

- 地址：`https://github.com/LAOYE1988/novel.git`
- 分支：`master`
- 可能因网络问题间歇性不可达

## 四、编码规范

### 4.1 通用规则
- 不添加任何注释，除非被明确要求
- 保持代码风格与现有文件一致
- 优先编辑已有文件，不新建文件
- 不主动创建文档（*.md）或 README
- 所有用户交互使用中文

### 4.2 Python 规范
- 使用标准库，不引入第三方包
- 文件编码统一 UTF-8
- 遵循已有文件的代码风格

### 4.3 前端规范
- 纯 HTML/JS/CSS，不引入框架
- API 调用走 `http://localhost:8899/api/`

## 五、项目管理

### 5.1 小说项目结构

每部小说的标准目录结构：
```

├── 大纲/         # 总纲.md、分卷.md、逐章细纲.md
├── 文稿/         # 第N章.txt
└── 设定集/       # 世界观.md、人物.json、伏笔.json、关系图.md、场景.md
```

### 5.2 当前激活项目

通过 `tools/novels_config.json` 管理，当前激活项目为 **奇幻大陆**。

### 5.3 敏感信息保护

- `.env` 文件含 DeepSeek API Key，已在 `.gitignore` 中排除
- 禁止在任何消息或日志中暴露 API Key

## 六、AI 智能体系统

| 类别 | 智能体 | 职责 |

| 基础设定 | 书风认知-投喂专员 | 文风记忆、同风格仿写 |

| 审核优化 | 金牌责编-润色 | 去AI味、质感提升 |
| 写作素材 | 智能选词词库助手 | 分类描写词汇 |

## 七、关键约束

1. **不覆盖本地代码**：拉取前必须先 stash 或确认工作区干净
2. **只操作 master 分支**：不创建或修改其他分支
3. **不自动处理冲突**：合并冲突交用户手动处理
4. **GitHub 可能不可达**：push/pull 失败时告知用户，不重复重试
5. **不出现在对话中**：本文档仅供 AI 代理和开发者阅读，不出现在用户消息中
