# 小说git操作

## Skill ID
NOV-GIT-001

## Skill 名称
小说git操作

## 归属模块
版本控制

## 能力简介
管理 novel 项目代码的版本提交、远程同步和历史回退。

## 触发条件
- 用户输入 `/提交 [说明]` —— 提交本地修改到远程仓库（说明可选，不填则仅标时间）
- 用户输入 `/下载` —— 从远程仓库拉取最新代码
- 用户输入 `/回滚` —— 回退到指定历史版本
- 前置依赖：当前工作目录为 `d:\trae\novel`，已关联远程仓库 `origin`

## 核心能力 & 功能点

### 1. 代码提交（/提交）
- 执行 `git add .` 暂存所有本地修改
- 生成提交信息规则：
  - 用户提供说明 → `说明 (yyyy-MM-dd HH:mm)`
  - 用户未提供说明 → `自动提交 (yyyy-MM-dd HH:mm)`
- 执行 `git commit -m` 写入含时间戳的提交记录
- 执行 `git push origin master` 推送至远程仓库
- 仅执行上传操作，不拉取远程代码

### 2. 代码下载（/下载）
- 执行 `git stash push` 暂存本地未提交修改，防止冲突
- 执行 `git pull --ff-only` 快进合并下载远程代码
- 快进失败时降级为 `git pull` 自动合并
- 合并完成后恢复本地暂存的修改

### 3. 版本回滚（/回滚）
- 输出最近 10 条提交历史供用户选择
- 支持软回滚 `git reset --soft`：保留本地修改，仅退版本指针
- 支持硬回滚 `git reset --hard`：彻底恢复至目标版本
- 硬回滚前必须获取用户二次确认

## 输出规范
- 操作结果以中文消息反馈，标注成功 / 失败
- 失败时输出具体错误原因，不吞没异常
- 版本历史按 `提交编号 + 提交说明 + 时间` 格式展示

## 禁用 / 边界
- 不执行 `git fetch` 或 `git pull` 以外的远程拉取操作
- 不自动合并冲突，冲突场景交由用户手动处理
- 不修改远程仓库的除 `master` 外的分支
