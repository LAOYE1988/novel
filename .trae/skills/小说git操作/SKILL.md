# 小说项目 Git 助手

处理 novel 项目的 Git 操作，通过三个中文命令控制：提交、下载、回滚。

---

## 命令一：`/提交 <说明>`

**用途：** 将本地的修改安全推送到远程 GitHub 仓库（只上传，不下载）。

**执行步骤：**
1. `git status` — 检查当前状态
2. `git add .` — 添加所有修改
3. `git commit -m "<说明> (yyyy-MM-dd HH:mm)"` — 提交到本地
4. `git push origin master` — 推送到远程仓库

**安全保证：** 不会执行 `pull` 或 `fetch`，不会被远程覆盖。

**示例：**
> 用户说：`/提交 第6章新增5000字`
> 提交信息：`第6章新增5000字 (2026-06-25 21:30)`

---

## 命令二：`/下载`

**用途：** 从远程 GitHub 仓库拉取最新代码到本地（只下载，不上传）。

**执行步骤：**
1. `git status --porcelain` — 检查是否有未提交的本地修改
2. 如果有未提交的修改：`git stash push -m "自动暂存"` — 先保护起来
3. `git pull --ff-only` — 优先快进合并
   - 如果失败，尝试 `git pull`（自动合并）
   - 如果还失败，提示用户手动处理冲突
4. 如果之前暂存了：`git stash pop` — 恢复本地修改

**安全保证：** 在本地的修改会被暂存保护后再拉取，不会被覆盖。

**示例：**
> 用户说：`/下载`
> 拉取远程最新代码到本地

---

## 命令三：`/回滚 [版本标识]`

**用途：** 将代码回退到历史版本。

**执行步骤：**

### 方案 A：用户指定具体版本
1. `git log --oneline -10` — 显示最近10条提交历史
2. 让用户确认要回滚到的版本
3. 询问用户的回滚方式：
   - **软回滚**（`git reset --soft`）：保留本地修改，只退版本指针（安全，推荐）
   - **硬回滚**（`git reset --hard`）：彻底回到旧版本，之后的所有修改消失
4. 用户确认后执行

### 方案 B：用户说 `/回滚` 没给版本
1. `git log --oneline -10` — 显示最近10条提交历史（含编号）
2. 让用户选择要回退到哪个版本
3. 同方案A第3-4步

**安全保证：** 硬回滚会丢失数据，必须让用户二次确认。优先推荐使用 `git revert` 创建反向提交（保留历史）。
